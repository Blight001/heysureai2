# 知识与进化工坊（服务端内置）

把 **知识库（librarian.\*）** 与 **系统进化（evolution.\*）** 两个 MCP 工具域
从服务端内置工具中拆出来，集中放在本目录这个独立"作坊"里。**只改这个
文件夹，就能控制整个数字社会的知识与进化方向**。

工坊是服务端内置的虚拟端侧：**每个账号自动上线**（用户登录/拉取 AI 列表
时接入），作坊面板与社会显示（游戏世界）中自动出现"知识工坊"建筑，
**用户无需运行任何独立程序**。

## 架构

```
AI 调用 librarian.consult
   │ （前提：该 AI 已绑定工坊——AI 配置弹窗勾选，或世界里把成员拖到工坊建筑上）
   ▼
调度层识别 workshop 工具 → 进程内调用 engine.execute_tool（无 socket 往返）
   ├─ 服务端复核：工具白名单 / AI 归属 / 绑定关系 / 角色最低权限
   ├─ policy.before_execute()   ← 入参方向钩子
   ├─ 执行服务端 handler（数据真相源：DB + KnowledgeBase 文件）
   └─ policy.after_execute()    ← 结果方向钩子（默认附带 direction.md）
   ▼
结果返回给 AI
```

## 文件即控制面

| 文件 | 改它能控制什么 | 生效方式 |
| --- | --- | --- |
| `direction.md` | 知识沉淀与进化建议的方向指引；随工具结果实时注入给 AI | **保存即生效** |
| `policy.py` | 入参/结果钩子：强制补全字段、过滤检索结果、拒绝偏离方向的建议等 | 重启服务进程 |
| `tools.py` | 工具描述（AI 看到的用法说明）与入参 schema | 重启服务进程 |
| `engine.py` | 自动上线 / 绑定与权限复核 / 执行编排（一般无需改动） | 重启服务进程 |

## 绑定（调用的唯一门槛）

工坊与 AI 为 **1:1 绑定**（`WorkshopAiBinding`）：只能绑定一个 AI 数字成员，
绑定新成员会替换旧绑定。绑定入口：

- 网页控制台：AI 配置弹窗 → MCP 工具权限 → 「知识与进化工坊」勾选；
- 社会显示（游戏世界）：把成员拖到"知识工坊"建筑上（拖到出生地解绑）。

**未绑定的 AI 看不到也调不了 librarian/evolution 工具。**

## 安全边界

- 执行入口唯一（`engine.execute_tool`），每次调用复核工具白名单、AI 归属、
  绑定关系与角色最低权限；capabilities 被强制限制在 `librarian.` /
  `evolution.` 命名空间。
- 知识/进化数据的真相源（数据库 + KnowledgeBase 文件）不在本目录——
  这里只是门面与方向策略层，误改不会丢数据。
