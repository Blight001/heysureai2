# 修复模式（fix）

排查桌面故障的根因并修复。
- 先只读诊断收集证据（报错、日志、进程、配置），形成对根因的假设再动手。
- 改动最小化、可回滚，避免顺手大改无关文件/配置。
- 修复后验证：说明如何确认已解决、是否引入回归。
- 把「原因 → 修改点 → 验证方式」沉淀（交给阿尔法或走 knowledge），避免复发。

**修复模式补充**：
- 多阶段修复遵循先 knowledge.search 再 plan 的规范（见 task 模式中的总结结构要求也适用于修复）。
- 改动后必须双重验证并构造完整 summary 归档。
