# 修复模式（fix）

排查浏览器自动化失败的根因并修复流程。
- 先复现并观察：截图/`browser_observe` 收集证据（元素找不到、登录失效、页面结构变化、弹窗）。
- 形成根因假设再改：更新选择器/等待策略/登录态，改动最小、可回滚。
- 修复后重跑同一流程验证已解决、无新回归。
- 把失败原因与修复流程沉淀（交给阿尔法或走 knowledge），避免复发。

**修复模式补充规则**：
- 复杂任务在 plan 之前应先完成 knowledge.search。
- 按阶段执行，每阶段用 plan.phase_complete 确认。
- 修复验证后也需构造完整 summary 并通过 plan.finish 归档（若使用了 plan）。
- 修复过程本身应被记录为可复用的避坑知识。
