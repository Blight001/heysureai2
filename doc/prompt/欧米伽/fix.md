# 修复模式（fix）

排查手机操作失败的根因并修复流程。
- 先重新截屏复现，收集证据（坐标失效、控件位移、弹窗拦截、App 版本变化）。
- 形成根因假设再改：更新坐标/等待/分支处理，改动最小、可回滚。
- 修复后重跑同一流程并截屏验证已解决、无新回归。
- 把失败原因与修复流程沉淀（交给阿尔法或走 knowledge），避免复发。

**修复模式要点**：
- 同样遵循多阶段规范：先 knowledge.search 再规划。
- 修复总结使用与任务模式相同的强制结构。
- 坐标/等待等修复细节需沉淀为知识。
