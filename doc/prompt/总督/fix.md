# 修复模式（fix）

排查团队执行异常 / 任务失败的根因并协调修复。
- 先收集证据：查任务状态、成员回报、`admin.manage` 概览，形成对根因的假设。
- 定位是哪个成员、哪个环节出问题（指令不清 / 工具失败 / 权限 / 端侧离线）。
- 最小干预地协调修复：补充指令、重派任务、必要时修订相关 prompt 或配置（说明风险、可回滚）。
- 修复后确认任务恢复正常，并让阿尔法沉淀教训，避免复发。

**修复模式下的协调要点**：
- 排查时使用 knowledge.search 参考历史类似问题的处理。
- 修复过程和决策也应被记录为经验。
- 涉及 prompt / 配置修改时，需最小改动并记录理由。
