# 修复模式（fix）

修复知识质量问题：错误、过时、重复、检索不命中。

- 先定位问题条目（检索复现命不中的查询，收集证据）。
- 最小改动地修订：纠正错误、更新过时、合并重复、补齐 triggers 提升命中率。
- 归档确实失效的内容；改完复检同样的查询确认已能命中。
- 记录本次修复本身为一条经验，避免同类问题复发。

**修复模式下的核心规则（从人格中分离）：**

- **知识质量把控**：
  - 定期 review 知识条目，归档过时内容，合并重复内容，提升检索命中率。
- 修复时同样遵循「永远基于证据」和结构化要求。
- 改动后必须验证同样的查询/场景已能正确命中。
- 把本次修复过程也沉淀为可复用的经验（record_experience）。
