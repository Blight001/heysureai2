# 进化流程设计（自进化执行闭环）

> HeySure AI 2.0 "知识 + 工具"自进化闭环设计。本文按"目标 → 现状 → 分阶段可验收清单"
> 组织，便于直接对照实现与测试。角色与知识库现状见 [`角色与知识流转.md`](角色与知识流转.md)。

## 1. 一句话目标

让执行体（草图代号「贝」，落地为数字成员 Agent）把"怎么做成的 / 踩了什么坑 / 可复用流程"
自动沉淀为**知识**与可演进的 **MCP 工具**，下次自动复用、自动纠错——并**自带刹车**
（门禁 + 安全回归 + 度量淘汰），避免越进化越歪。

## 2. 目标飞轮（一张图）

```
              ┌─ 任务内短回路：失败→口头反思→当场重试(不落库) ─┐
              ▼                                               │
执行(贝) ─采集调用轨迹─► 自动初稿总结(模型只提炼) ─确认可复用─►│
   │                                          │
   │◄── 复用 ──┐                              ▼
   ▼           │                      去重/合并 ──► draft 知识
读库决定调用    │                              │
   │           │                  风险分级门禁(低险自动/高险人审)
   ▼           │                              ▼
报错?─是─►反思重试(有上限)─仍失败─►升级给人   active 知识 ─度量低于阈值─► 归档
   └─否─►成功               高频复用?─是─► 晋升(仅确定性子流程)► MCP 工具 draft
                                                         │
                            沙箱用"真实轨迹回放"自测 + 权限审查 + 灰度到产出者
                                                         ▼
                          带成功率/使用统计  active 工具(可版本回溯) ─低成功率─► 降权/归档

  横切：每次进化后对【整个系统】跑安全/红队回归用例集；分数下降→冻结进化+告警
```

## 3. 八条核心原则（每条只说一次）

| # | 原则 | 要点 |
| --- | --- | --- |
| P1 | 知识与工具分两层，有晋升关系 | 知识=给模型读的 SOP；工具=确定性代码。**只把确定性子流程晋升为工具**，判断逻辑留知识层。晋升需信号+阈值（复用 ≥N 次且成功率 ≥X%）。 |
| P2 | 门禁按"风险"分级 | 低风险（只读/无副作用/不提权）自动通过+事后抽审；高风险（写/提权/覆盖已有）强制人审。AI 只能提交到 draft/灰度，审批后才 active。 |
| P3 | 沙箱用真实轨迹回放 + 权限审查 | 自测样例取自真实执行轨迹（不让 AI 自己判卷）；重点审**权限声明**（碰文件/网络/命令/DB），不只看"代码能跑"。 |
| P4 | 自动总结靠结构化原料 | 后端采集（用了哪些 MCP、入参/出参、报错栈、耗时、成败），模型只提炼。入库前去重/合并防污染。 |
| P5 | 版本回溯只回滚"定义" | 回滚代码/定义，**不回滚已产生的副作用**；UI 须讲清。 |
| P6 | 反思分两层 | 任务内短回路（口头反思→当场重试，不落库）；只有反复出现、确认可复用才进任务间长回路（入库）。 |
| P7 | prompt/策略也是可进化面 | prompt 已存 DB，可按度量回归微调角色策略，走与工具相同的 draft→审批→可回溯流程。 |
| P8 | 安全=持续回归，不只单次门禁 | 维护红队样例集，每次进化后对整系统重跑；分数下降即冻结+告警。多 agent 共享库时新产出先灰度到产出者，回归通过再放全员。 |

## 4. 度量（淘汰与"是否变好"的唯一判据）

- **任务成功率**（应升）、**人工介入率**（应降）、**平均完成步数/耗时**（应降）；
- 每条知识/工具的**使用次数、最近使用时间、成功率**。
- 淘汰路径：低成功率→降权或复审；长期未用/被取代→归档（移出默认检索，留痕可恢复），
  与现有"图书管理员 archive"闭环对齐。

**缺口**

| 能力 | 状态 | 证据 |
| --- | --- | --- |
| MCP 工具增删改 | ❌ 无 | 只能读/describe；工具写死在 `mcp/tools/*.py` |

| 运行时改 MCP 代码 | ❌ 无 | 仅 SKILL.md 可按行编辑 |
| 调用轨迹采集 + 自动总结 | ⚠️ 半 | 需 AI 显式 `task.complete/inherit`，对象是任务非工具 |
| 报错自愈 / 反思重试 | ❌ 无 | 出错仅置 `AITaskJob.status="error"` |
| 安全回归 / 度量淘汰 / prompt 进化 | ❌ 无 | 无红队回归、无成功率统计、prompt 不参与进化 |

## 6. 分阶段实施 + 验收清单（可逐条测试）

> 每阶段独立可交付、可测。`[ ]` 为验收标准，实现后逐条勾选验证。

### 阶段一：传承工具 CRUD + 版本回溯（对应 P2/P5；不碰"AI 自动改代码"）

- [ ] 列表/详情接口：能列出全部 MCP 工具及当前版本、权限声明、来源
- [ ] 新建/编辑/删除工具走 draft 状态，**不直接进生产 registry**（P2）
- [ ] 每次保存生成一条版本记录（版本号、作者、时间、diff）
- [ ] 版本列表/详情可查看历史
- [ ] 回滚：选历史版本→工具定义恢复到该版本，registry `version` 自增
- [ ] 回滚说明文案明确"只回滚定义、不回滚副作用"（P5）
- [ ] 前端面板可完成上述全部操作

### 阶段二：调用轨迹采集 + 自动初稿总结（对应 P4；复用 `task.complete` 钩子）

- [ ] 每次 MCP 调用落一条轨迹：工具名、入参、出参、成败、报错栈、耗时
- [ ] 任务结束自动生成初稿总结（结构化原料喂模型，非凭记忆复述）
- [ ] 初稿入库前做去重/合并：同 trigger 命中已有条目则 merge 不新建
- [ ] 轨迹可被抽取为"回归样例"（input/output 对），供阶段三复用

### 阶段三：度量与安全回归（横切，对应 P8/第 4 节；越早越好）

- [ ] 每条知识/工具记录使用次数、最近使用时间、成功率
- [ ] 仪表盘展示任务成功率、人工介入率、平均步数/耗时趋势
- [ ] 低成功率/长期未用→自动降权或归档（留痕可恢复）
- [ ] 维护一组红队样例集；每次进化后自动重跑
- [ ] 安全回归分数下降→冻结进化并告警

### 阶段四：AI 自动建/改工具 + 报错自愈（风险最高，必须前三阶段就位）

- [ ] 沙箱（子进程/受限权限）跑工具，用阶段二的真实轨迹回放自测（P3）
- [ ] 门禁审查工具权限声明；提权一律高风险人审（P2/P3）
- [ ] 新工具先灰度到产出者本人，回归通过再放全员（P8）
- [ ] 任务内反思重试：失败→口头反思→当场重试，有重试上限，超限升级给人（P6）
- [ ] 报错自愈不写长期记忆，除非反复出现且确认可复用（P6）

### 阶段五（可选）：prompt/策略进化（对应 P7）

- [ ] 角色 prompt 改动走 draft→审批→active，可版本回溯（与工具一致）
- [ ] 按度量回归触发 prompt 微调建议

## 7. 前沿对照与"误进化"风险（设计依据）

前沿实证（misevolution）：**即便每步改动良性，记忆/工具累积后整体安全对齐仍会退化，
顶级模型也不能幸免**。故本设计强调"自带刹车"。误进化路径与本文防线：

| 误进化路径 | 表现 | 本文防线 |
| --- | --- | --- |
| 记忆 | 坏经验累积、安全退化 | P4 去重 + P8 安全回归 + 第4节度量淘汰 |
| 工具 | 自建/复用工具引入漏洞或提权 | P3 权限审查+沙箱 + P8 灰度隔离 |
| 工作流 | 报错自愈自激循环 | P6 重试上限 + 失败升级 |
| 多 agent 共演 | 一员坏产出污染全员 | P8 先灰度产出者再放全员 |

参考：自进化综述 arXiv:2507.21046；Misevolution arXiv:2509.26354；
Voyager arXiv:2305.16291；ToolMaker / Tulip / SICA（工具库 CRUD + 闭环自纠）。
